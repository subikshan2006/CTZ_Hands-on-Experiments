# Task 11 — Entry & Exit Criteria per TDLC Phase

| TDLC Phase | Entry Criteria | Exit Criteria |
|---|---|---|
| Unit Testing | Code for the module is complete and compiles/builds successfully; unit test cases are written | All unit tests pass; code coverage meets the agreed threshold (e.g. 80%); no critical defects open |
| Integration Testing | All required unit-tested modules are available and integrated into a build; integration test plan is approved | All integration test cases executed; interfaces between modules verified; no open critical/high defects |
| System Testing | Application is fully integrated and deployed to the test environment; system test plan is approved | All system test cases executed; end-to-end flows verified; defect count below the agreed threshold; no open critical/high defects |
| Acceptance Testing (UAT) | System testing is complete and signed off; UAT environment mirrors production; business/admin users are available | Business stakeholders formally accept the system; all acceptance criteria met; sign-off document obtained |
