# Task 9 — V-Model Diagram

```
Requirements                                   Acceptance Testing
      \                                              /
       System Design                    System Testing
              \                          /
               Architecture Design   Integration Testing
                      \              /
                       Module Design  Unit Testing
                              \      /
                               Coding
```

**Left side (development):** Requirements -> System Design -> Architecture Design ->
Module Design -> Coding.

**Right side (testing):** Acceptance Testing -> System Testing -> Integration Testing ->
Unit Testing, converging back up from Coding at the bottom vertex.

Each development phase on the left has a directly corresponding testing phase on the
right, drawn at the same height, forming the "V" shape with Coding at the bottom point.
