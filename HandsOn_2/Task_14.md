# Task 14 — QA's Role in Agile Ceremonies

| Ceremony | QA's Role |
|---|---|
| Sprint Planning | Defines clear, testable Acceptance Criteria for each story (e.g. for "Create Course" story, specifies success and validation-error scenarios) so the team agrees on "done" upfront. |
| Daily Standup | Raises blocking issues, such as a broken test environment or a flaky endpoint blocking test execution, so they get resolved quickly. |
| Sprint Review | Demonstrates tested functionality to stakeholders, showing not just the happy path but also how validation/error cases behave, and gathers feedback. |
| Retrospective | Contributes process improvement ideas, e.g. proposing that API contract tests run in CI before merge to catch integration defects earlier next sprint. |
